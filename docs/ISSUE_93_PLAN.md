# Issue #93 실행 계획 (사진 모아 보내기)

## 목표
- 기존 단일 이미지 업로드에서 다중 선택 업로드를 지원한다.
- UX는 기존과 호환되도록 유지한다.

## 단계
1. 파일 input에 `multiple` 속성 추가
2. 파일 선택 핸들러에서 `FileList`를 배열로 처리
3. 1장 선택 시 기존 편집(preview/crop/rotate) 플로우 유지
4. 2장 이상 선택 시 편집 없이 순차 업로드 + 순차 메시지 전송
5. 업로드 완료 토스트 추가

## 리스크/대응
- 대량 업로드 시 지연: 순차 업로드로 안정성 우선
- 중복 파일명: 랜덤 suffix 추가
- 실패 UX: 토스트 에러 표준화

## 완료 기준
- 여러 장 선택 시 모두 업로드되고 채팅에 연속 전송됨
- 기존 단일 업로드 편집 플로우는 그대로 동작
